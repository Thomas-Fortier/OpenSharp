## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

Derived from the OpenSharp Constitution v1.0.0. Confirm each gate or record a justified
deviation in Complexity Tracking.

- **I. Code Quality**: Plan avoids inline comments unless justified; lint/format gates apply.
- **II. Documentation**: All new public members will carry XML doc comments; generated docs
  remain part of the build.
- **III. Performance**: Performance-sensitive work declares measurable goals in Technical
  Context; regressions block release.
- **IV. Testing (NON-NEGOTIABLE)**: Plan includes unit tests (Moq) and, where feasible,
  Reqnroll system tests; the design keeps the ≥80% coverage gate achievable.
- **V. Separation of Concerns & DI**: Core logic targets a dedicated class library;
  dependencies are injected, not constructed inline.
- **VI. Cross-Platform**: Design is cross-platform by default; any platform restriction is
  documented and justified in the spec.
- **VII. Clarification Over Assumption**: Open questions are raised with the user rather than
  assumed; unavoidable defaults are recorded explicitly.
